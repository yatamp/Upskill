How Large Language Models Actually Work

An LLM is a neural network trained to predict the next token in a sequence. That's it — but emergent capabilities arise at scale.


Tokens

Models don't see words — they see tokens. A token ≈ 4 chars. "Hello world" = 2 tokens. Claude's context window is measured in tokens (e.g., 200K tokens ≈ 150K words ≈ a long book).

Transformer Architecture

Attention mechanism: each token attends to all other tokens. Deeper layers = higher-level concepts. Training: predicts next token on billions of text examples. Fine-tuning + RLHF makes it helpful/safe.

Temperature & Sampling

Temperature 0 = deterministic (always picks highest probability token). Temperature 1 = more random/creative. For code: use 0. For brainstorming: use 0.7–1.0.

Context Window

Everything the model can "see" at once — system prompt + conversation history + your message. Claude 3.5+ supports 200K tokens. Models have no memory outside the context window (unless you add RAG/memory systems).

Hallucinations

Models generate plausible-sounding text — they can confidently state false facts. Always verify critical outputs. Reduce with: grounding (RAG), low temperature, explicit instructions to say "I don't know."

Prompt Engineering

The craft of writing inputs that get reliable outputs. Key techniques: role assignment, chain-of-thought, few-shot examples, output format specification, XML tags for structure.

Claude Models — Current Lineup (2025)

Anthropic releases model families. Current generation: Claude 4.5 / 4.6.


Claude Opus 4.6

claude-opus-4-6

Most capable. Best for complex reasoning, coding, analysis. Slower and more expensive. Use when quality matters most. Ideal for autonomous agents.

Claude Sonnet 4.6

claude-sonnet-4-6

Best balance of capability and speed/cost. Great for most tasks — chat, coding, analysis, pipelines. This is what Claude Code runs on.

Claude Haiku 4.5

claude-haiku-4-5-20251001

Fastest and cheapest. Use for high-volume, simpler tasks — classification, summarization, quick lookups. Excellent for real-time applications with latency requirements.


Rule of thumb: Start with Sonnet. Step up to Opus if quality isn't sufficient. Step down to Haiku if cost/latency is the bottleneck.

Claude API — Building With It

The Anthropic API lets you call Claude programmatically. Available in Python, TypeScript/Node, and via HTTP.


Install SDK:

# Python
pip install anthropic

# Node.js
npm install @anthropic-ai/sdk

Basic call — Python:

import anthropic

client = anthropic.Anthropic(api_key="your-api-key")  # or set ANTHROPIC_API_KEY env var

message = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    system="You are a DevOps expert. Be concise.",   # System prompt
    messages=[
        {"role": "user", "content": "Explain what a Kubernetes pod is in 2 sentences."}
    ]
)

print(message.content[0].text)

Multi-turn conversation — Node.js:

import Anthropic from '@anthropic-ai/sdk';

const client = new Anthropic();  // reads ANTHROPIC_API_KEY from env

const messages = [];

async function chat(userMessage) {
  messages.push({ role: 'user', content: userMessage });

  const response = await client.messages.create({
    model: 'claude-sonnet-4-6',
    max_tokens: 2048,
    system: 'You are a helpful DevOps assistant.',
    messages,
  });

  const reply = response.content[0].text;
  messages.push({ role: 'assistant', content: reply });
  return reply;
}

// Usage
console.log(await chat('What is CI/CD?'));
console.log(await chat('Give me an example with GitHub Actions.'));

Streaming — for real-time output:

const stream = await client.messages.stream({
  model: 'claude-sonnet-4-6',
  max_tokens: 1024,
  messages: [{ role: 'user', content: 'Write a Dockerfile for a Node.js app' }],
});

for await (const event of stream) {
  if (event.type === 'content_block_delta') {
    process.stdout.write(event.delta.text);
  }
}

Tool Use (Function Calling)

Tool use lets Claude call functions in your code — it decides when and with what arguments to call them. You execute them and return results. This is how AI agents work.


import anthropic

client = anthropic.Anthropic()

tools = [
    {
        "name": "get_deployment_status",
        "description": "Get the current deployment status of a service",
        "input_schema": {
            "type": "object",
            "properties": {
                "service_name": {
                    "type": "string",
                    "description": "The name of the service to check"
                },
                "environment": {
                    "type": "string",
                    "enum": ["dev", "staging", "production"],
                    "description": "The environment to check"
                }
            },
            "required": ["service_name", "environment"]
        }
    }
]

def get_deployment_status(service_name, environment):
    # Your actual implementation — call Kubernetes, AWS, etc.
    return {"status": "healthy", "version": "v2.1.3", "pods": 3}

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    tools=tools,
    messages=[
        {"role": "user", "content": "Is the auth-service healthy in production?"}
    ]
)

# Claude may return a tool_use block
if response.stop_reason == "tool_use":
    for block in response.content:
        if block.type == "tool_use":
            # Execute the tool
            result = get_deployment_status(**block.input)
            # Send result back to Claude
            final = client.messages.create(
                model="claude-sonnet-4-6",
                max_tokens=1024,
                tools=tools,
                messages=[
                    {"role": "user", "content": "Is the auth-service healthy in production?"},
                    {"role": "assistant", "content": response.content},
                    {"role": "user", "content": [
                        {"type": "tool_result", "tool_use_id": block.id, "content": str(result)}
                    ]}
                ]
            )
            print(final.content[0].text)

Prompt Caching — Reduce Costs 90%

If you repeat a large system prompt or document across many API calls, Anthropic caches the prompt and charges only 10% of input tokens on cache hits.

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    system=[
        {
            "type": "text",
            "text": "You are a DevOps expert...\n\n" + large_runbook_text,
            "cache_control": {"type": "ephemeral"}   # Mark for caching
        }
    ],
    messages=[{"role": "user", "content": "How do I roll back a deployment?"}]
)
# First call: full price. Subsequent calls within 5 min: 90% cheaper.
When to use: Any system prompt >1024 tokens that you reuse. Long documents/codebases you query repeatedly. Cache lasts 5 minutes (refreshed on each cache hit).

MCP — Model Context Protocol

MCP is an open standard (published by Anthropic) that lets AI models connect to external tools, data sources, and services through a unified interface. Think of it as USB-C for AI — one standard, any tool.


How it works:

Host (Claude Desktop / Claude Code / your app)
    ↕ JSON-RPC over stdio or HTTP+SSE
MCP Server (a process exposing tools, resources, prompts)
    ↕ native calls
External system (filesystem, database, GitHub, Slack, AWS, etc.)

Three primitives MCP servers expose:

PrimitiveWhat it isExample
ToolsFunctions the model can call (like function calling)create_file, run_query, send_slack
ResourcesRead-only data the model can access (like files/URIs)file:///project/src/, db://users
PromptsReusable prompt templates the user/model can invoke/review-pr, /explain-error

Configure MCP servers in Claude Desktop or Claude Code:

// ~/.claude/settings.json (Claude Code) or Claude Desktop config
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/you/projects"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_TOKEN": "ghp_..." }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres", "postgresql://localhost/mydb"]
    }
  }
}

Build your own MCP server — Python (FastMCP):

from mcp.server.fastmcp import FastMCP

mcp = FastMCP("DevOps Tools")

@mcp.tool()
def get_pod_status(namespace: str, pod_name: str) -> dict:
    """Get the status of a Kubernetes pod"""
    import subprocess, json
    result = subprocess.run(
        ["kubectl", "get", "pod", pod_name, "-n", namespace, "-o", "json"],
        capture_output=True, text=True
    )
    return json.loads(result.stdout)

@mcp.tool()
def tail_logs(service: str, lines: int = 50) -> str:
    """Get the last N log lines from a service"""
    import subprocess
    result = subprocess.run(
        ["kubectl", "logs", f"deployment/{service}", "--tail", str(lines)],
        capture_output=True, text=True
    )
    return result.stdout

if __name__ == "__main__":
    mcp.run()

Then configure it:

{
  "mcpServers": {
    "devops-tools": {
      "command": "python",
      "args": ["/path/to/your_mcp_server.py"]
    }
  }
}
Real power: Once your MCP server is running, Claude Code can call your Kubernetes cluster, query your database, check your monitoring system — all through natural language. "Show me the logs for auth-service and check if there are any errors in the last 100 lines."

Building AI Agents

Agents are AI systems that can take sequences of actions to complete a task. Claude is well-suited for agentic tasks because of its large context window, tool use, and ability to plan.


Simple agent loop pattern:

import anthropic

client = anthropic.Anthropic()

def run_agent(task: str, tools: list, tool_executor):
    """
    A simple agent loop: run until Claude stops requesting tools.
    """
    messages = [{"role": "user", "content": task}]

    while True:
        response = client.messages.create(
            model="claude-opus-4-6",   # Use Opus for agents — best reasoning
            max_tokens=4096,
            tools=tools,
            messages=messages,
        )

        # Add Claude's response to history
        messages.append({"role": "assistant", "content": response.content})

        if response.stop_reason == "end_turn":
            # Claude is done — extract final text
            for block in response.content:
                if hasattr(block, 'text'):
                    return block.text

        if response.stop_reason == "tool_use":
            # Execute all requested tools
            tool_results = []
            for block in response.content:
                if block.type == "tool_use":
                    result = tool_executor(block.name, block.input)
                    tool_results.append({
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": str(result)
                    })

            messages.append({"role": "user", "content": tool_results})

Agent best practices:
• Add a max_iterations limit — prevent infinite loops
• Log every tool call — agents are hard to debug without traces
• Use Opus for reasoning-heavy agents, Sonnet for lighter ones
• Give Claude a clear stopping condition in the system prompt
• Test with simple tasks before complex multi-step ones

Hands-On Project: Build a CI Failure Explainer

A GitHub Action that calls Claude to explain CI failures in plain English and comments on the commit.

1
Set up API key 
# In your GitHub repo → Settings → Secrets → Actions
# Add: ANTHROPIC_API_KEY = your key from console.anthropic.com
2
Create the explainer script 
# .github/scripts/explain_failure.py
import anthropic, sys, os

def explain_failure(logs: str) -> str:
    client = anthropic.Anthropic()
    response = client.messages.create(
        model="claude-haiku-4-5-20251001",   # Use Haiku — fast and cheap for this
        max_tokens=512,
        system="You are a DevOps expert. Given CI failure logs, explain the root cause in 2-3 sentences and suggest a fix. Be direct.",
        messages=[
            {"role": "user", "content": f"CI failure logs:\n\n{logs[:8000]}"}
        ]
    )
    return response.content[0].text

if __name__ == "__main__":
    logs = sys.stdin.read()
    print(explain_failure(logs))
3
Create the GitHub Actions workflow 
# .github/workflows/explain-failures.yml
name: Explain CI Failures

on:
  workflow_run:
    workflows: [CI]
    types: [completed]

jobs:
  explain:
    if: github.event.workflow_run.conclusion == 'failure'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Get failure logs
        id: logs
        run: |
          # Download logs from the failed workflow
          gh run view ${{ github.event.workflow_run.id }} --log-failed > failure.log 2>&1 || true
          echo "logfile=failure.log" >> $GITHUB_OUTPUT
        env:
          GH_TOKEN: ${{ github.token }}

      - name: Explain failure with Claude
        run: |
          pip install anthropic -q
          EXPLANATION=$(cat failure.log | python .github/scripts/explain_failure.py)
          echo "EXPLANATION<> $GITHUB_ENV
          echo "$EXPLANATION" >> $GITHUB_ENV
          echo "EOF" >> $GITHUB_ENV
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}

      - name: Comment on commit
        run: |
          gh api repos/${{ github.repository }}/commits/${{ github.event.workflow_run.head_sha }}/comments \
            -f body="🤖 **CI Failure Analysis**

          ${{ env.EXPLANATION }}

          [View failed run](${{ github.event.workflow_run.html_url }})"
        env:
          GH_TOKEN: ${{ github.token }}

Check Your Understanding

Q1. You're querying a 50-page technical runbook via the API in a loop (100 calls/day). How do you reduce costs?

Show answer

Use prompt caching. Add "cache_control": {"type": "ephemeral"} to the system message containing the runbook. First call charges full price; subsequent calls within 5 minutes charge 10% of input tokens. 100 calls/day → ~90% cost reduction on that portion.

Q2. What is the difference between MCP Tools and MCP Resources?

Show answer

Tools are callable functions with side effects (write, query, execute) — the model decides when to call them. Resources are read-only data the model can access (like files or DB records) — they're surfaced as context rather than function calls. Tools = actions, Resources = data.

Q3. Why use Claude Opus for an agentic task but Haiku for classifying support tickets?

Show answer

Agents require multi-step planning, tool selection, error recovery, and complex reasoning — Opus's superior reasoning justifies the higher cost when each run might cost $0.50 but save hours of work. Ticket classification is a simple, high-volume task — thousands per day. Haiku costs ~20x less per call and is fast enough for real-time response. Match model capability to task complexity.

Q4. What should go in a CLAUDE.md file?

Show answer

Project-specific persistent context that Claude should always know: tech stack & versions, coding conventions & style rules, architecture overview (key directories, data flows), commands to run (tests, lint, build), constraints (what NOT to do), and any domain knowledge specific to the project. It's Claude's onboarding doc — write it like you'd brief a new engineer.
