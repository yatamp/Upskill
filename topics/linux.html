Linux Basics

Level: Beginner → Intermediate  ·  Estimated: 3–4 days  ·  Every server you touch runs Linux

How Linux Works — The Architecture

Kernel

The core — manages CPU, memory, devices, system calls. User programs never touch hardware directly; they ask the kernel via syscalls (read, write, fork, exec, etc.).

Shell

A user-space program that reads commands and executes them. bash, zsh, sh are all shells. When you type a command, the shell fork()s a child process and exec()s the binary.

Everything is a File

Linux's core philosophy. Regular files, directories, devices (/dev/sda), sockets, pipes — all accessed via the same file I/O interface. /proc and /sys expose kernel state as files.

Processes

Every running program is a process with a PID. Each process has: a parent (PPID), file descriptors, memory space, environment variables, a working directory. Init (PID 1) is the ancestor of all processes.

File Descriptors

Numbers representing open files/connections. Every process starts with: 0 = stdin, 1 = stdout, 2 = stderr. When you redirect (2>&1), you're remapping FDs.

systemd

Modern init system (PID 1 on most distros). Manages services (systemctl start/stop/enable), logging (journalctl), targets (runlevels), and dependencies between services.

File Permissions — The Part Everyone Gets Wrong

Every file has permissions for three classes: owner (u), group (g), others (o).

-rwxr-xr--  1  pyatam  staff  4096  Apr 9  script.sh
 ↑↑↑↑↑↑↑↑↑
 │└──┬──┘└──┬──┘└──┬──┘
 │  owner  group  others
 │  rwx    r-x    r--
 type (- = file, d = dir, l = symlink)

Permission bits:

r = 4read
w = 2write
x = 1execute

Octal notation: rwxr-xr-- = 754 (7=rwx, 5=r-x, 4=r--)


CommandEffect
chmod 755 script.shrwxr-xr-x — owner full, group+others read+execute
chmod 644 config.confrw-r--r-- — owner read+write, others read-only
chmod +x script.shAdd execute bit for all classes
chmod -R 755 /var/wwwRecursively set permissions
chown pyatam:staff file.txtChange owner to pyatam, group to staff
chown -R www-data /var/wwwRecursively change owner to www-data
umask 022Default permission mask — new files get 644, dirs get 755

SUID / SGID / Sticky

SUID (4xxx): Run file as owner, not caller. passwd has SUID — runs as root even when called by normal user.
Sticky bit (1xxx): On directories — only the owner can delete their own files. Set on /tmp.

sudo

Run commands as another user (usually root). Config in /etc/sudoers. Never log in as root directly — use sudo for auditability. sudo -i = root shell. sudo -u www-data cmd = run as www-data.

ACLs

Extended permissions beyond owner/group/others. setfacl -m u:nginx:rx /var/app grants nginx user access without changing group. getfacl file to view.

Shell Scripting — The Essentials

#!/bin/bash
set -euo pipefail   # ALWAYS put this at the top
# -e: exit on error   -u: error on unset var   -o pipefail: catch pipe failures

# Variables
NAME="world"
echo "Hello, $NAME"

# Command substitution
DATE=$(date +%Y-%m-%d)
FILES=$(ls /etc/*.conf | wc -l)

# If / else
if [[ -f "/etc/nginx/nginx.conf" ]]; then
    echo "nginx config found"
elif [[ -d "/etc/nginx" ]]; then
    echo "nginx dir exists but no config"
else
    echo "nginx not installed"
fi

# Comparison operators
# Strings: == != -z (empty) -n (not empty)
# Numbers: -eq -ne -lt -gt -le -ge
# Files: -f (file) -d (dir) -x (executable) -r (readable)

# Loops
for i in {1..5}; do
    echo "Step $i"
done

for file in /var/log/*.log; do
    echo "Processing: $file"
    wc -l "$file"
done

# While loop
COUNT=0
while [[ $COUNT -lt 3 ]]; do
    echo "Count: $COUNT"
    ((COUNT++))
done

# Functions
deploy() {
    local SERVICE=$1    # local = scoped to function
    local ENV=${2:-staging}   # default value
    echo "Deploying $SERVICE to $ENV"
    # ... actual deploy logic
}
deploy "auth-service" "production"

# Error handling
check_health() {
    local URL=$1
    local STATUS
    STATUS=$(curl -s -o /dev/null -w "%{http_code}" "$URL")
    if [[ "$STATUS" != "200" ]]; then
        echo "ERROR: $URL returned $STATUS" >&2
        return 1
    fi
    echo "OK: $URL"
}

# Arrays
SERVICES=("auth" "api" "worker" "scheduler")
for svc in "${SERVICES[@]}"; do
    echo "Checking $svc..."
done
echo "Total services: ${#SERVICES[@]}"

# Read input from file or stdin
while IFS= read -r line; do
    echo "Processing: $line"
done < "/path/to/file"

# Trap for cleanup
cleanup() {
    echo "Cleaning up..."
    rm -f /tmp/deploy.lock
}
trap cleanup EXIT   # runs on any exit, including errors

Write your own service unit:

# /etc/systemd/system/myapp.service
[Unit]
Description=My Application
After=network.target postgresql.service
Requires=postgresql.service

[Service]
Type=simple
User=appuser
WorkingDirectory=/opt/myapp
ExecStart=/opt/myapp/bin/server --port 8080
ExecReload=/bin/kill -HUP $MAINPID
Restart=on-failure
RestartSec=5
StandardOutput=journal
StandardError=journal
Environment=NODE_ENV=production
EnvironmentFile=/etc/myapp/env    # load env vars from file

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable --now myapp

# *       *    *             *      *

@reboot       /opt/scripts/startup.sh       # On every reboot

# Best practices
# 1. Redirect output or you'll get email spam
*/5 * * * * /script.sh >> /var/log/script.log 2>&1

# 2. Use full paths (cron has minimal PATH)
0 * * * * /usr/bin/python3 /opt/myapp/task.py

# 3. Use flock to prevent overlapping runs
*/5 * * * * flock -n /tmp/task.lock /opt/scripts/task.sh

#!/bin/bash
set -euo pipefail

LOG_FILE="/var/log/bootstrap.log"
exec > >(tee -a "$LOG_FILE") 2>&1

echo "=== Bootstrap started: $(date) ==="

# Check running as root
if [[ $EUID -ne 0 ]]; then
    echo "ERROR: Must run as root" >&2
    exit 1
fi

# Update system
apt-get update -qq
apt-get upgrade -y -qq

# Install essentials
apt-get install -y -qq \
    curl wget git vim htop \
    build-essential \
    ufw fail2ban \
    jq unzip

# Create deploy user
if ! id deploy &>/dev/null; then
    useradd -m -s /bin/bash deploy
    usermod -aG sudo deploy
    mkdir -p /home/deploy/.ssh
    chmod 700 /home/deploy/.ssh
    echo "Created deploy user"
fi

# Configure UFW firewall
ufw --force reset
ufw default deny incoming
ufw default allow outgoing
ufw allow 22/tcp comment "SSH"
ufw allow 80/tcp comment "HTTP"
ufw allow 443/tcp comment "HTTPS"
ufw --force enable

# Harden SSH
cat >> /etc/ssh/sshd_config << 'EOF'
PermitRootLogin no
PasswordAuthentication no
X11Forwarding no
MaxAuthTries 3
ClientAliveInterval 300
ClientAliveCountMax 2
EOF
systemctl restart sshd

# Configure fail2ban
systemctl enable --now fail2ban

echo "=== Bootstrap complete: $(date) ==="

Check Your Understanding

Q1. What does chmod 640 secrets.env mean? Who can read it?

Show answer

6 = rw- (owner can read+write), 4 = r-- (group can read only), 0 = --- (others cannot access). Only the file owner and members of the file's group can read it. Nobody outside the group can read, write, or execute. Good for config files with credentials.

Q2. Your script works fine manually but fails in cron. What are the most likely causes?

Show answer

(1) PATH — cron has minimal PATH. Use full paths like /usr/bin/python3. (2) Working directory — cron starts in / by default, not your home dir. Use absolute paths. (3) Environment variables — cron doesn't load your shell profile. Set vars explicitly or source them. (4) No output capture — errors may be silently swallowed. Add 2>&1 and log to file.

Q3. What is the difference between kill -15 and kill -9?

Show answer

SIGTERM (-15) is a polite request to terminate — the process can catch this signal and do cleanup (close connections, flush buffers, write PID file). Always try this first. SIGKILL (-9) is immediately terminated by the kernel — the process cannot catch, block, or ignore it. No cleanup happens. Use -9 only if -15 fails after a reasonable wait.

Q4. What does 2>&1 do and where would you use it?

Show answer

Redirects file descriptor 2 (stderr) to wherever fd 1 (stdout) currently points. So command > output.log 2>&1 sends both stdout and stderr to output.log. Critical in cron jobs and scripts where you want to capture all output. Order matters: 2>&1 > file is wrong (stderr goes to terminal, stdout to file); > file 2>&1 is correct.
